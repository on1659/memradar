1. 문제 정의

핵심 문제: AI의 행동을 사용자 성격으로 오귀인

현재 시스템은 edit, write, bash 같은 AI가 알아서 쓰는 도구의 사용량으로 사용자 성격을 판단합니다. 그 결과 거의 모든 사용자가 "만능 빌더(EWM)"으로 수렴합니다.

축현재 측정문제
Style (R/E)AI 도구 비율 (Read vs Execute)AI 자율 선택이라 사용자 의도 무관 → 거의 E
Scope (D/W)도구+프로젝트+모델 다양성Claude Code 기본 도구가 7-10개라 항상 높음 → 거의 W
Rhythm (M/S)평균 세션시간 / 30분대부분 세션이 15분+라 절대값 기준 초과 → 거의 M

설계 원칙

5대 원칙
  1. 사용자가 직접 통제하는 행동에서만 측정할 것
  2. toolsUsed는 성격 축 측정에서 완전 배제
  3. 축 간 상관을 최소화 (독립 차원)
  4. 절대 임계값 금지 → 시그모이드 정규화 기반 상대적 분류
  5. 3축 8유형 뼈대 유지 (UI/콘텐츠 변경 최소화)

세션 종류 분류

단순 횟수가 아닌 세션 구성 비율로 리듬을 판단한다.

종류기준의미
Quick20분 미만빠른 질문, 간단한 수정
Standard20~60분일반적인 작업 세션
Deep60분 이상깊은 몰입, 대규모 작업
왜 세션 횟수가 아니라 종류 비율인가?

하루 10번 쓰더라도 7번이 Deep이면 마라토너. 하루 2번만 쓰더라도 둘 다 Quick이면 스프린터. 횟수가 아닌 작업 스타일을 직접 반영.

측정 수식
// 세션 종류 비율
quickCount = sessions where duration < 20min
deepCount  = sessions where duration >= 60min
sessionTypeScore = deepCount / (deepCount + quickCount)
// 둘 다 0이면 (Standard만) → 0.5 (중립)

// 시간대 엔트로피 (Shannon entropy)
hourlyEntropy = -sum(p * log2(p)) for each hour
concentration = 1 - (hourlyEntropy / log2(24))

rhythmScore = normalize(medianDuration)     * 0.4
            + normalize(concentration)       * 0.3
            + normalize(sessionTypeScore)    * 0.3

>= 0.5 → M (마라토너)  |  < 0.5 → S (스프린터)
마라토너(M)가 하는 말
"한번 앉으면 1-2시간은 기본이야"
스프린터(S)가 하는 말
"틈틈이 짧게 하는 게 맞아"

/** 시그모이드 정규화: value를 0~1 범위로 매핑 */
function sigmoid(value: number, midpoint: number, steepness: number): number {
  if (!Number.isFinite(value)) return 0.5
  return 1 / (1 + Math.exp(-steepness * (value - midpoint)))
}

/** cwd 경로에서 프로젝트 루트를 추출 */
function extractProject(cwd: string): string {
  const parts = cwd.replace(/\\/g, '/').split('/').filter(Boolean)
  return parts.slice(0, Math.min(parts.length, 4)).join('/')
}

/** 배열의 중앙값 계산 */
function median(arr: number[]): number {
  if (arr.length === 0) return 0
  const sorted = [...arr].sort((a, b) => a - b)
  const mid = Math.floor(sorted.length / 2)
  return sorted.length % 2 ? sorted[mid] : (sorted[mid - 1] + sorted[mid]) / 2
}

export function computePersonality(sessions: Session[], stats: Stats): PersonalityResult {
  const defaultAxes: Record<AxisKey, AxisScore> = {
    style:  { label: ['설계자', '탐험가'], value: 0.5 },
    scope:  { label: ['한우물', '유목민'], value: 0.5 },
    rhythm: { label: ['마라토너', '스프린터'], value: 0.5 },
  }

  if (sessions.length === 0 || stats.totalMessages < 3) {
    return { type: 'EWS', axes: defaultAxes, ...TYPE_DEFS.EWS }
  }

  // ═══════════════════════════════════════════════════
  // 축 1: 대화 방식 — 설계자(A) vs 탐험가(E)
  // 사용자 메시지의 길이, 턴 수, 토큰 비율로 측정
  // ═══════════════════════════════════════════════════

  // 사용자 메시지만 추출
  const userMessages = sessions.flatMap(s =>
    s.messages.filter(m => m.role === 'user')
  )
  const avgMsgLen = userMessages.length > 0
    ? userMessages.reduce((sum, m) => sum + m.text.length, 0) / userMessages.length
    : 100

  // 세션당 평균 사용자 턴 수
  const turnsPerSession = sessions.length > 0
    ? sessions.reduce((sum, s) => sum + s.messageCount.user, 0) / sessions.length
    : 8

  // 출력/입력 토큰 비율
  const tokenRatio = stats.totalTokens.input > 0
    ? stats.totalTokens.output / stats.totalTokens.input
    : 5

  const styleValue =
    sigmoid(avgMsgLen, 100, 0.02) * 0.4 +            // 메시지 길수록 → A
    (1 - sigmoid(turnsPerSession, 8, 0.2)) * 0.3 +    // 턴 적을수록 → A
    sigmoid(tokenRatio, 5, 0.3) * 0.3              // AI출력 많을수록 → A

  // ═══════════════════════════════════════════════════
  // 축 2: 작업 범위 — 한우물(D) vs 유목민(W)
  // cwd 기반 프로젝트 집중도로 측정. toolsUsed 배제.
  // ═══════════════════════════════════════════════════

  const projectIds = sessions
    .map(s => s.cwd ? extractProject(s.cwd) : '')
    .filter(Boolean)
  const uniqueProjects = new Set(projectIds)

  // 주당 고유 프로젝트 수 (기간 보정)
  const firstTime = new Date(sessions[0].startTime).getTime()
  const lastTime = new Date(sessions[sessions.length - 1].endTime).getTime()
  const activeWeeks = Math.max((lastTime - firstTime) / (7 * 24 * 3600000), 1)
  const projectsPerWeek = uniqueProjects.size / activeWeeks

  // 프로젝트 전환 빈도
  let switchCount = 0
  for (let i = 1; i < projectIds.length; i++) {
    if (projectIds[i] !== projectIds[i - 1]) switchCount++
  }
  const switchRate = projectIds.length > 1
    ? switchCount / (projectIds.length - 1)
    : 0

  // 최다 프로젝트 집중도
  const projectCounts: Record<string, number> = {}
  for (const pid of projectIds) projectCounts[pid] = (projectCounts[pid] || 0) + 1
  const topProjectShare = projectIds.length > 0
    ? Math.max(...Object.values(projectCounts)) / projectIds.length
    : 1

  const scopeValue =
    sigmoid(projectsPerWeek, 2.5, 0.8) * 0.4 +       // 프로젝트 많을수록 → W
    sigmoid(switchRate, 0.3, 5) * 0.4 +               // 전환 잦을수록 → W
    (1 - sigmoid(topProjectShare, 0.6, 5)) * 0.2     // 집중도 낮을수록 → W

  // ═══════════════════════════════════════════════════
  // 축 3: 작업 리듬 — 마라토너(M) vs 스프린터(S)
  // 세션 중앙값 + 시간 엔트로피 + 세션 종류 비율
  // ═══════════════════════════════════════════════════

  const durations = sessions
    .map(s => (new Date(s.endTime).getTime() - new Date(s.startTime).getTime()) / 60000)
    .filter(d => d > 0 && d < 1440)  // 0 초과, 24시간 미만
  const medianDuration = median(durations)

  // 시간대 엔트로피 (stats.hourlyActivity 활용)
  const hours = stats.hourlyActivity
  const totalActivity = hours.reduce((a, b) => a + b, 0)
  let hourlyEntropy = 0
  if (totalActivity > 0) {
    for (const h of hours) {
      if (h > 0) {
        const p = h / totalActivity
        hourlyEntropy -= p * Math.log2(p)
      }
    }
  }
  const concentration = 1 - (hourlyEntropy / Math.log2(24)) // 0=분산, 1=집중

  // 세션 종류 비율: Quick(<20분) vs Deep(>=60분)
  const quickCount = durations.filter(d => d < 20).length
  const deepCount = durations.filter(d => d >= 60).length
  const sessionTypeScore = (quickCount + deepCount) > 0
    ? deepCount / (quickCount + deepCount)
    : 0.5  // Standard만 있으면 중립

  const rhythmValue =
    sigmoid(medianDuration, 45, 0.06) * 0.4 +        // 길수록 → M
    sigmoid(concentration, 0.5, 4) * 0.3 +           // 집중적일수록 → M
    sigmoid(sessionTypeScore, 0.5, 4) * 0.3         // Deep 많을수록 → M

  // ═══════════════════════════════════════════════════
  // 유형 결정
  // ═══════════════════════════════════════════════════

  const axes: Record<AxisKey, AxisScore> = {
    style:  { label: ['설계자', '탐험가'], value: styleValue },
    scope:  { label: ['한우물', '유목민'], value: scopeValue },
    rhythm: { label: ['마라토너', '스프린터'], value: rhythmValue },
  }

  // TypeCode ���핑 (기존 코드와 동일한 문자열 유지)
  // style: A(설계자) → 기존 'E' 위치, E(탐험가) → 기존 'R' 위치
  // 이유: TYPE_DEFS에서 EDM="장인대장장이" 등을 그대로 유지하기 위해
  const s = styleValue >= 0.5 ? 'E' : 'R'   // A(설계자)→E, E(탐험가)→R
  const d = scopeValue >= 0.5 ? 'W' : 'D'   // W(유목민), D(한우물)
  const r = rhythmValue >= 0.5 ? 'M' : 'S'  // M(마라토너), S(스프린터)
  const type = `${s}${d}${r}` as TypeCode

  return { type, axes, ...TYPE_DEFS[type] }
}
TypeCode 매핑 주의

기존 TYPE_DEFS에서 EDM = "장인 대장장이", RDM = "심해 잠수부"로 정의되어 있습니다. 새 축에서 설계자(A)가 기존 E 위치, 탐험가(E)가 기존 R 위치에 매핑되어야 유형 이름이 맞습니다. TypeCode 문자열 자체(EDM, RDS 등)는 변경하지 않습니다.

축 라벨 변경

PersonalityView.tsx와 PersonalitySlide.tsx에서 축 라벨은 axes.style.label을 그대로 렌더링합니다. computePersonality()에서 반환하는 label을 ['설계자', '탐험가'] 등으로 바꾸면 UI에 자동 반영됩니다.

UI 변경이 필요 없는 이유

PersonalityView.tsx의 바 UI는 axes[key].label[0], axes[key].label[1], axes[key].value를 동적으로 렌더링합니다. label 문자열을 바꾸면 표시가 자동으로 바뀝니다.
